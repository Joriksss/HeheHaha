<h1>Краткая инструкция по работе с базой данных</h1>

    <p>
        Это краткая инструкция по работе с базой данных для проекта на Streamlit.
        Здесь описаны основные моменты, которые нужно учитывать при создании,
        заполнении и проверке базы данных.
    </p>

    <h2>Заполнение базы данных</h2>

    <p>
        Базу данных можно заполнить вручную через DBeaver.
    </p>

    <p>
        Для удобной вставки SQL-кода можно использовать сочетания клавиш:
        сначала <b>Ctrl + C</b> для копирования, затем <b>Ctrl + Shift + V</b>
        для вставки в базу данных.
    </p>

    <p>
        В DBeaver можно выделять нужные части SQL-кода отдельно:
        создание таблиц, <code>SELECT</code>-запросы, <code>INSERT</code>-запросы
        и другие команды.
    </p>

    <h2>Важный порядок создания таблиц</h2>

    <p>
        Таблицы нужно создавать в правильном порядке, потому что между ними
        есть связи через внешние ключи.
    </p>

    <p>
        Сначала создаются небольшие справочные таблицы:
    </p>

    <ul>
        <li><code>roles</code></li>
        <li><code>units</code></li>
        <li><code>manufacturers</code></li>
        <li><code>suppliers</code></li>
        <li><code>categories</code></li>
        <li><code>pickup_points</code></li>
    </ul>

    <p>
        После этого создаются основные таблицы:
    </p>

    <ul>
        <li><code>users</code></li>
        <li><code>products</code></li>
        <li><code>orders</code></li>
        <li><code>order_items</code></li>
    </ul>

    <p>
        Такой порядок нужен из-за внешних ключей. Например, товар связан
        с категорией, производителем, поставщиком и единицей измерения.
        Поэтому сначала должны существовать справочные таблицы, а потом уже
        таблица товаров.
    </p>

    <h2>Порядок данных в SQL и Python</h2>

    <p>
        Порядок полей в базе данных, SQL-запросах и Python-коде должен быть одинаковым.
    </p>

    <p>
        Если в базе или в <code>INSERT</code>-запросе указан один порядок полей,
        то в Python данные нужно передавать в такой же последовательности.
    </p>

    <pre><code>INSERT INTO products(
    article, name, unit_id, price, supplier_id, manufacturer_id, category_id,
    discount, stock_quantity, description, photo_url
)
VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)</code></pre>

    <p>
        Тогда в Python данные должны идти в таком же порядке:
    </p>

    <pre><code>(
    article,
    name,
    unit_id,
    price,
    supplier_id,
    manufacturer_id,
    category_id,
    discount,
    stock_quantity,
    description,
    photo_url
)</code></pre>

    <p>
        Если порядок перепутать, приложение может выдавать ошибки или неправильно
        добавлять данные.
    </p>

    <h2>Проверка</h2>

    <p>
        После создания и заполнения базы нужно проверить, что таблицы реально есть
        внутри базы и приложение подключается именно к нужному файлу.
    </p>

    <p>
        Если приложение пишет, что таблицы нет, значит Python может брать базу
        из другой папки.
    </p>

    <h2>Изображения</h2>

    <p>
        Изображения товаров можно заменить на свои. По умолчанию можно использовать
        файл <code>picture.png</code>.
    </p>

    <h2>Итог</h2>

    <p>
        Нужно проверить путь к базе, порядок создания таблиц, порядок полей
        в SQL-запросах и Python-коде, а также наличие данных. После этого
        приложение должно работать корректно.
    </p>
