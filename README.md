Краткая инструкция по работе с базой данных

В проекте база данных создаётся и заполняется отдельно. Главное — следить за тем, чтобы приложение подключалось именно к нужному файлу базы, а порядок таблиц и полей в SQL совпадал с порядком в Python-коде.

Заполнение базы данных

Базу можно заполнять вручную через DBeaver.

Самый удобный способ:

Сначала копируем нужный SQL-код через Ctrl + C, затем вставляем его в DBeaver через Ctrl + Shift + V. После этого можно выделить нужный участок кода и выполнить его.

В DBeaver можно отдельно запускать разные части SQL-кода: создание таблиц, добавление данных, SELECT-запросы и другие команды.

Порядок создания таблиц

Таблицы нужно создавать в правильном порядке, потому что в базе используются связи через внешние ключи.

Сначала создаются небольшие справочные таблицы: роли, единицы измерения, производители, поставщики, категории и пункты выдачи.

После этого создаются основные таблицы: пользователи, товары, заказы и состав заказа.

Это важно, потому что, например, товар связан с категорией, поставщиком, производителем и единицей измерения. Поэтому сначала должны существовать эти справочные таблицы, а уже потом таблица товаров.

Порядок данных в SQL и Python

Порядок полей должен быть одинаковым везде: в таблице базы данных, в SQL-запросе и в Python-коде.

Если в SQL-запросе сначала идёт артикул, потом название, потом единица измерения, цена, поставщик и так далее, то в Python значения должны передаваться в таком же порядке.

Если порядок перепутать, данные могут вставиться неправильно. Например, цена может попасть не в поле цены, а в другое поле. Из-за этого приложение может выдавать ошибки, хотя на первый взгляд кажется, что проблема в другом месте.

Проверка базы

После создания базы нужно проверить, что таблицы действительно есть внутри файла базы данных.

Также важно проверить, что Python подключается именно к нужной базе. Если приложение берёт базу из другой папки, оно может писать, что таблицы нет, хотя в другой базе она уже создана.

Поэтому перед запуском нужно убедиться, что путь к базе данных указан правильно.

Изображения товаров

Изображения товаров можно заменить на свои. По умолчанию можно использовать стандартную картинку picture.png.

Потом при необходимости можно поменять картинки, названия и остальные данные под свой вариант.

Итог

Перед запуском проекта нужно проверить базу, путь к ней, порядок создания таблиц и порядок полей в SQL-запросах. Если всё сделано правильно, приложение должно нормально работать.
