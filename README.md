<h1>Модуль 1. Разработка базы данных</h1>

<h2>Третья нормальная форма</h2>

<p>
Третья нормальная форма нужна для того, чтобы в базе данных не было лишнего повторения данных.
Если какое-то значение часто повторяется, его лучше вынести в отдельную таблицу.
</p>

<p>
В моём задании основная сущность — вакансия. У вакансии есть компания, регион и специализация.
Эти данные могут повторяться в разных вакансиях, поэтому они вынесены в отдельные таблицы.
</p>

<h2>Основные таблицы</h2>

<pre><code>region
- id
- region
- code

company
- company_id
- company_name
- email
- INN
- KPP
- OGRN

specialization
- id
- specialization

users
- id
- role
- full_name
- login
- password

vacancy
- id
- name
- data_create
- region_id
- company_id
- specialization_id
- salary_from
- salary_to</code></pre>

<h2>Связи между таблицами</h2>

<p>
Основная таблица — <code>vacancy</code>. Именно она хранит внешние ключи на справочники.
</p>

<pre><code>vacancy.region_id → region.id
vacancy.company_id → company.company_id
vacancy.specialization_id → specialization.id</code></pre>

<p>
То есть в таблице вакансий не хранится текстом название региона, компании и специализации.
Вместо этого хранятся их идентификаторы.
</p>

<h2>Почему это соответствует 3НФ</h2>

<ul>
  <li>Регионы вынесены в отдельную таблицу.</li>
  <li>Компании вынесены в отдельную таблицу.</li>
  <li>Специализации вынесены в отдельную таблицу.</li>
  <li>В таблице вакансий нет лишнего дублирования текста.</li>
  <li>Связи между таблицами сделаны через внешние ключи.</li>
</ul>

<h2>Важно про автозаполнение</h2>

<p>
В итоговой версии автозаполнение идентификаторов можно не использовать вручную.
В DBeaver значения первичных ключей можно вводить самостоятельно или оставить обработку на стороне базы данных,
в зависимости от настроек таблицы.
</p>

<p>
На практике при ручном заполнении таблиц я могу сам указать нужные значения <code>id</code>,
чтобы потом правильно связать вакансии с регионами, компаниями и специализациями.
</p>

<h2>Лайфхак для вставки данных в DBeaver</h2>

<p>
Чтобы быстро вставить данные в таблицу, можно подготовить строки в Excel или другом табличном редакторе,
скопировать нужный диапазон и вставить его в таблицу DBeaver.
</p>

<pre><code>1. Открыть таблицу в DBeaver.
2. Выделить нужные данные в Excel.
3. Нажать Ctrl + C.
4. В DBeaver нажать Ctrl + Shift + V.
5. Проверить вставленные данные.
6. Сохранить изменения.</code></pre>

<p>
Такой способ удобен, если нужно быстро перенести данные без написания большого количества
<code>INSERT</code>-запросов.
</p>

<h2>Краткий итог</h2>

<p>
База данных была разделена на несколько связанных таблиц. Повторяющиеся данные вынесены в справочники,
а основная таблица <code>vacancy</code> хранит ссылки на них через внешние ключи.
Такая структура подходит под требования третьей нормальной формы.
</p>
